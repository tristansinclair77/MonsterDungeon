# Quick Start Guide - Testing Player Movement

## How to Test the Player Movement Feature

### Step 1: Build and Run
1. Open the solution in Visual Studio
2. Press **F5** (or click Start Debugging)
3. The game window should appear

### Step 2: Navigate to Combat Screen
1. Press the **` (backtick)** key to open the Debug Menu
2. In the Debug Menu, find the "Screen Selection" dropdown
3. Select **"Combat Screen"** from the dropdown
4. The combat grid should now be visible

### Step 3: Test Player Movement
1. You should see a green circular sprite with the letter "P" on the grid
   - It should be near the bottom-center of the grid (Row 8, Column 4)

2. **Test Left Movement**:
   - Press the **Left Arrow** key once
   - The "P" sprite should move one tile to the left

3. **Test Right Movement**:
   - Press the **Right Arrow** key once
   - The "P" sprite should move one tile to the right

4. **Test Boundaries**:
   - Keep pressing Left Arrow until you reach the left edge
   - Try pressing Left Arrow again - the sprite should stay at the edge
   - Move to the right edge and test the same

5. **Test Key Hold Prevention**:
   - Press and HOLD the Left Arrow key
   - The sprite should move only once (not continuously)
   - Release and try again

## Expected Behavior

### ? Correct Behavior
- Player moves exactly one tile per key press
- Player stops at grid edges (columns 0-7)
- Holding key down does NOT cause continuous movement
- Movement is instant and responsive

### ? Incorrect Behavior (Report if seen)
- Player doesn't move at all
- Player moves continuously when holding key
- Player moves off the grid
- Player sprite is not visible
- Movement is delayed or sluggish

## Troubleshooting

### Problem: Combat Screen is not showing
**Solution**: 
- Press ` to open Debug Menu
- Make sure "Combat Screen" is selected in the dropdown
- If still not visible, check the Output window for errors

### Problem: Keyboard input not working
**Solution**:
- Click on the combat grid area to ensure it has focus
- Try pressing Tab to cycle through controls
- Restart the application

### Problem: Player sprite "P" is not visible
**Solution**:
- Check that the Debug Menu is NOT covering the grid
- Press ` to hide the Debug Menu if open
- Verify the CombatViewModel is initialized (check debug output)

### Problem: Build fails
**Solution**:
- Clean the solution (Build ? Clean Solution)
- Rebuild (Build ? Rebuild Solution)
- Check Output window for specific error messages
